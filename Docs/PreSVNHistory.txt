================================================================================

VIS: Historical update information from v1.0 to Release 7.0.3

================================================================================

This file records known changes to files in VIS from v1.0 and release 7.0.3
before the project was placed was placed under version control with Subversion.

There are two sections:

  1: Releases: Lists all the releases of CodeSnip and notes which files were
     added, updated or deleted in each release.

  2: Files: Lists all source code and development tools for which history is
     available and provides details of changes to these files.

================================================================================

v1.1 of 03 May 1998
--------------------------------------------------------------------------------

v5.0 of 04 Aug 2002
--------------------------------------------------------------------------------
+ Now uses version information access code from an external DLL rather than the
  TPJVersionInfo component. This code can display version info from files that
  are not internally consistant.
+ Where there is an internal integrity problem within version information this
  is now flagged by displayed affected translations in red.
+ All required DLLs are loaded dynamically and the if the DLL is critical the
  application is terminated gracefully if the DLL is not present.
+ All version information string information is now displayed rather just the
  predefined strings. The string information can now be sorted on the string
  name.
+ Added display of Creation date in fixed file information and provided option
  to the fixed file info structure information.
+ Moved File | Preferences sub menu to its own Options menu.
+ Added ability to manage the new context menu handler.
+ Added ability to open file from command line.
+ Added ability to optionally run only one instance of application when called
  from context menu handler.
+ Gave main form window class a unique name for search purposes.
+ Added custom exception dialog box.
+ Now has version information provided from a separate resource file rather than
  being auto-created by Delphi.
+ Help file updated re new features and a "What's new" topic added.
+ Program's own version information now supplied via .vi file rather than
  automatically generated by Delphi.

--------------------------------------------------------------------------------

                       TDDabblerDLLLoader and made it place [Common Files]\
                       DelphiDabbler on search path rather than [Common Files]\
                       PJSoft.

                         command line application.
