Dealing With Errors and Inconsistencies

Version Information Spy can handle version information data that breaks the resource's data format rules. It can even display some code that the Windows API will fail on. It silently reads and displays the data in these cases.

The program can also cope with version information that breaks the semantic rules set down by Microsoft. It detects some of these and draws them to the user's attention. In particular the program flags up the following conditions:

  1. Where there is translation information present but there are no string tables.
  2. Where there are string tables but no translation data.
  3. Where there are both string tables and translation data but the string table's locale information does not match the translation data.

These cases are highlighted in the following ways:

  1. Highlighting a problem translation in the translation drop down list.
  2. Placing a brief description of the error above the drop down list.
  3. Displaying a button above the drop down list that, when clicked, displays a report of the problem in the Translation Inconsistency Explanation dialog box. This report also suggests how to fix the problem.
  4. Noting problems in the text and HTML reports.

The first three methods noted above can be configured and disabled or enabled by the user in the Display Options dialog box.

Finally, the program can generate RC source code for the version information that either fixes the problems or notes the problems in comments in the code.
