About Fixed File Information

Fixed file information is made up of several integer valued fields, as follows:

File Version

Specifies the version number for the file.

Product Version

Specifies the version number of the product of which the file is part.

File Flags Mask

This bit mask specifies which elements of the File Flags- field (below) are valid. Only if an element is present in File Flags Mask is the corresponding element of File Flags valid. Version Information Spy can display either the numeric value of this item (in hexadecimal) or can described it fully – use the Describe file flags check box on the Fixed File Info tab of the Display Options dialog box to choose the display format.

File Flags

Specifies the Boolean attributes of the file in a bitmask.

Flag Purpose
Debug File may contain debugging information or have debugging features enabled.
Pre-release This is a development version of the file.
Patched File has been patched.
Private build File was not built using standard release procedures. The String Information should contain a PrivateBuild string.
Dynamically-created The version information was created dynamically rather than using the VERSIONINFO resource statement.
Special build This is a variation on a standard build. The String Information should contain a SpecialBuild string.

The display format used for File Flags depends on the Display Options setting noted in the description of File Flags Mask.

Operating System

Specifies the operating system for which this file was designed. The value displayed describes both the base and target operating systems.

File Type

Specifies the general type of the file. Further information about the types of some files may be given by the File Sub Type field (below). Supported file types are:

  • Application
  • Dynamic-link library (DLL)
  • Device driver*
  • Font*
  • Static-link library
  • Virtual device*

Any other file type will be flagged as unknown. File types marked with an asterisk have further information provided by the File Sub Type field.

File Sub Type

For certain file types, File Sub Type provides additional information to that given by the File Type field. The following table lists the file types for which further information is available and details what information is provided:

File type Additional information
Device driver
The type of driver is specified. It can be one of:
  • Communications driver
  • Printer driver
  • Keyboard driver
  • Language driver
  • Display driver
  • Mouse driver
  • Network driver
  • System driver
  • Installable driver
  • Sound driver
If the type is not recognised or not specified then the Unknown driver type is displayed.
Font
The type of font is specified:
  • Raster font
  • Vector font
  • TrueType font
If the font type is not recognised or not specified then the Unknown font type is displayed.
Virtual device The virtual-device identifier from the virtual-device control block is displayed as a hexadecimal value.

Creation Date

The file's creation date may be included in the version information, although this is rarely used and is usually set to zero (Version Information Spy displays zero values as "No date specified"). This field is not displayed by default – check the Display creation date check box on the Fixed File Info tab of the Display Options dialog box to show the field.

Fixed File Info Structure Information

The fixed file information's structure signature and version number can optionally be displayed. The signature should be FEEF04BD (hex) and the structure version should be 1.0 in order to be confident that Version Information Spy has interpreted the structure correctly. This information is not displayed by default – check the Display structure information check box on the Fixed File Info tab of the Display Options dialog box to enable this feature.
