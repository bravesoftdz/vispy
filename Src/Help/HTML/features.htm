Main Features

The key features of Version Information Spy are:

  • Version information can be read from 16 and 32 bit executables (.exe, .dll, .ocx etc) and from 32 bit binary resource files.
  • Version information that contains information in more than one language is supported.
  • Reports describing a file's version information can be produced in text, XML and HTML format.
  • Binary version information can be de-compiled into the equivalent RC source code and any errors and inconsistencies in the original code can optionally be fixed.
  • Version information that departs from the Microsoft standard (and some that defeat the Windows API calls) are handled. Any inconsistencies in version information can be flagged up. Reports of errors and suggestions for fixes can be displayed.
  • There is a command line application that can produce text reports or decompiled source code.
  • Version Information Spy integrates with the Windows shell to provide two highly customisable shell extensions:
    1. A context menu handler that launches Version Information Spy to display version info for a selected executable or resource file.
    2. A property sheet extension that provides extended version information in a selected file's property sheet.
  • The configurable user interface supports XP themes.
