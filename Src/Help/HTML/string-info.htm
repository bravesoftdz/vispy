About String Information

String information is stored in Name/Value pairs. A different set of string information can exist for each "translation" in the version information resource. Version Information Spy displays all the string names and values defined in each translation.

Microsoft have predefined the meaning of some string names as follows:

String Name Description
Comments Custom information to be used for diagnostic purposes.
CompanyName Name of company that produced the file.
FileDescription Describes the purpose of the file.
FileVersion A text representation of the file's version number.
InternalName The internal name of the file if any. If there is no internal name the name of the file without extension is used.
LegalCopyright Copyright notices that apply to the file.
LegalTrademarks Trademarks that apply to the file.
OriginalFileName Original name of the file.
PrivateBuild Gives information about the private build of a file. The "private build" element of the file flags should be provided. See About Fixed File Information for details of file flags.
ProductName Name of the the product with which the file is distributed.
ProductVersion A text representation of the version number of the product with which the file is distributed.
SpecialBuild States how this file differs from the standard version of the file. The "special build" element of the file flags should be provided. About Fixed File Information for details of file flags.

Other string names can be (are often are) used. The program displays all the strings, not just the predefined ones. Non-standard string names can be highlighted and the highlight colour customised. Highlighting is configured on the String Info tab of the Display Options dialog box.
