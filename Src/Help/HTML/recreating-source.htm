Recreating RC Source Code

Version Information Spy can recreate, or at least estimate, the resource source code that was used to create a version information resource.

To do this open a file in the program to display its version information. Now select the Report | Source Code menu option. This will display the decompiled source in the Resource Source Code dialog box. You can copy this source to the clipboard or save it to disk. A resource compiler such as Borland's BRCC32 will be able to compile this code into a binary resource file.

If the version information has inconsistencies the program can also create source code that fixes the problems. Use the Report | Corrected Source Code menu option to do this. The result is displayed in the Resource Source Code dialog box.

The command line version of Version Information Spy can also be used to recreate a RC source file from a program's version information. See the command line program topic for an example of how to do this.
