Explorer Extension Configuration Dialog Box

This dialog box permits the Explorer shell extensions to be enabled, disabled and configured. The dialog is displayed by selecting the Options | Explorer Extensions menu option.

The dialog box has two tabs – basic and advanced. The basic tab can be used for straightforward configuration of the extensions, while the advanced tab is used for more refined "tweaking".

Basic tab

This tab contains four check boxes that are used to set up the shell extensions and the behaviour of Version Information Spy when working with the extensions. A description of each check box follows.

  • Enable Explorer context menu extension
    Use this check box to enable or disable the context menu handler. Enabling the handler registers it for use with all the file extensions listed on the Advanced tab. This handler adds a Version Information menu item to Explorer's context menu which opens the selected file in Version Information Spy.
  • Enable Explorer property sheet tab
    This check box enables or disables the property sheet tab extension. Enabling the property sheet tab registers it for use with all the file extensions listed on the Advanced tab. This extension adds a Version Extra tab to a file's property sheet and displays version information in it.
  • Explorer extensions use a single application instance
    The context menu handler extension activates Version Information Spy to display version information for a selected file while the property sheet extension can activate Version Information Spy when the Advanced button is clicked. You can configure these extensions to re-use an open instance of Version Information Spy (checked) or to always start a new instance (unchecked).
  • Ask user whether to register extension for new file extensions
    When Version Information Spy opens a file which has an extension that is not known the to explorer extensions it can offer to register the extension with them. Check this box to use this facility.

The Enable Explorer context menu extension and Enable Explorer property sheet tab check boxes can have an indeterminate state. In this case consult the Advanced tab to see which file extensions are registered.

Advanced tab

This tab lists the types of file containing version information that are known to the program. Some file types are pre-defined while others may have been added to the list by the user. There are two check boxes next to each file extension. Check the first check box (column CM) to register the file extension with the contect menu handler. Check the second box (column PS) to register it with the property sheet extension.

You can use the Select All button to check all the boxes or use Clear All to clear them all. This is the same as checking or clearing both the Enable Explorer context menu extension and Enable Explorer property sheet tab check boxes on the Basic tab.

New file extensions can be added to the list manually by clicking the Add button and selecting the required file extension from the Add Extension dialog box. The newly added file extension appears in the list. You need to tick the related check boxes to register the extension with the context menu or property sheet extension.

File extensions can be deleted from the list by highlighting the extension and clicking the Delete button. Note that the .dll, .exe and .res extensions are required and cannot be deleted, although they can be hidden from the shell extensions by clearing the appropriate check boxes.

Other controls

Once selections have been made choose the OK button to confirm selections or the Cancel button to abort the changes.

The dialog box remembers the tab that was in use when it is closed and displays the same tab again when the dialog is next opened.
